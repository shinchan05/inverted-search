📌 Overview

Inverted Search is based on the concept of an inverted index, where instead of storing data in a forward manner (document → words), we store it as:

word → list of documents

This makes searching extremely fast and efficient, especially for large datasets.


🚀 Features
🔎 Search for a word across multiple files
📂 Index multiple documents
⚡ Fast lookup using inverted indexing
📊 Displays list of files where the word appears
🧠 Efficient data structure usage

⚙️ How It Works
Read multiple input files
Break content into words
Store each word in a data structure
Map word → list of file names
Search queries return matching files instantly
